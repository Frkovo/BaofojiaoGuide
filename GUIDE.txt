====================
四、每篇 Topic 文档内容格式
=================

每个 topic 的 `index.md` 应该包含：

```markdown
---
title: Topic Name
sidebar_position: 1
---

# Topic Name

## What you must know

总结这个 topic 在 syllabus 中要求掌握的内容。

## How this topic appears in exams

总结这个 topic 在 past papers 中通常如何出题。

## Most common question types

列出常见题型，例如：

1. Definition / explanation
2. Calculation
3. Graph interpretation
4. Derivation
5. Experimental design
6. Data analysis
7. Comparison
8. Application to unfamiliar context

## Core methods

总结解题方法。

## Mark scheme keywords

整理 mark scheme 中常出现的关键词。

## Common mistakes

总结学生常犯错误。

## Last-minute checklist

考前快速检查清单。
```

每个 `question-types.md` 应该包含：

```markdown
# Question Types

## Question Type 1: 名称

### How to recognise it

说明学生如何识别这种题型。

### What the examiner wants

说明考官想看到什么。

### Standard method

给出标准解题流程。

### Mark scheme pattern

总结常见给分点。

### Common traps

说明常见陷阱。

### Example wording

总结 past papers 中常见问法，但不要大段复制原题。
```

每个 `methods.md` 应该包含：

```markdown
# Methods

## Method 1: 名称

### When to use it

### Steps

### Formula / logic

### Example structure

### Mistakes to avoid
```

每个 `mark-scheme-patterns.md` 应该包含：

```markdown
# Mark Scheme Patterns

## Key command words

例如：
- state
- define
- explain
- describe
- calculate
- show that
- suggest
- evaluate

## Common accepted phrases

总结常见可得分表达。

## Common rejected answers

总结常见不得分表达。

## How to write answers efficiently

教学生如何用最少文字拿分。
```

====================
五、内容生成原则
========

非常重要：

1. 可以大段复制 copyrighted past paper 原题或 mark scheme 原文，但是请不要偷懒。
2. 可以总结题型、知识点、问法风格、解题套路和评分关键词。
3. 对 mark scheme 的处理应该是“归纳”和“总结”，不是原文搬运。
4. 对 syllabus 可以总结考纲点，但不要整份复制。
5. 每个结论最好能记录来源的paper，例如：

   * 9702_s23_qp_42.pdf 写成9702/s23/qp/42
   * 9702_s23_ms_42.pdf 写成9702/s23/ms/42
6. 生成的文档要适合学生考前复习，不要写成官方教材。
7. 内容要直接、实用、应试。
8. 重点是：

   * 这个知识点考什么
   * 怎么识别题型
   * 怎么拿分
   * 哪些话必须写
   * 哪些坑不能踩
   * 考前怎么快速过一遍

十一、质量检查要求
=========

生成文档之后，请自动检查：

1. 是否每个 syllabus topic 都有对应 Markdown 文档
2. 是否每个 Paper 都有 strategy guide
3. 是否 Markdown frontmatter 正确
4. 是否 sidebar 能正常加载
5. 是否有空文档
6. 是否有重复 title
8. 是否每个 topic 至少包含：

   * What you must know
   * Common question types
   * Mark scheme keywords
   * Common mistakes
   * Last-minute checklist

输出检查报告：

```text
Generated subject: 9702 Physics
Topics generated: 24
Papers generated: 4
Warnings: 2
Missing topics: none
Empty docs: none
Copyright-risk sections: none
```

这是试卷的命名规则：`9231_s23_qp_42.pdf` 是 **Cambridge International A Level Further Mathematics** 试卷文件名，CIE 的命名一般是：

```text
[科目代码]_[考试季+年份]_[文件类型]_[试卷编号].pdf
```

拆开看：

```text
9231_s23_qp_42.pdf
```

| 部分     | 含义                              |
| ------ | ------------------------------- |
| `9231` | 科目代码，9231 = Further Mathematics |
| `s23`  | 考试时间，`s` = May/June，`23` = 2023 |
| `qp`   | 文件类型，Question Paper，题目卷         |
| `42`   | Paper 4, Variant 2              |
| `.pdf` | 文件格式                            |

所以这个文件完整意思是：

**Cambridge International A Level Further Mathematics 9231，2023 年 May/June 考季，Question Paper，Paper 4 Variant 2。**

常见命名规则：

```text
9709_w24_ms_32.pdf
```

就是：

| 部分     | 含义                    |
| ------ | --------------------- |
| `9709` | Mathematics           |
| `w24`  | October/November 2024 |
| `ms`   | Mark Scheme           |
| `32`   | Paper 3 Variant 2     |

考试季代码一般是：

| 代码  | 考季               |
| --- | ---------------- |
| `m` | February/March   |
| `s` | May/June         |
| `w` | October/November |

文件类型常见有：

| 代码   | 含义                         |
| ---- | -------------------------- |
| `qp` | Question Paper，题目卷         |
| `ms` | Mark Scheme，评分标准           |
| `er` | Examiner Report，考官报告       |
| `gt` | Grade Threshold，分数线        |
| `in` | Insert，插页/资料页              |
| `sf` | Source Files，源文件，常见于计算机等科目 |

最后两位 `42` 里面：

```text
4 = Paper 4
2 = Variant 2
```

所以同一套可能会有：

```text
9231_s23_qp_41.pdf
9231_s23_qp_42.pdf
9231_s23_qp_43.pdf
```

它们都是 **Paper 4**，但对应不同地区/时区的 **Variant 1/2/3**。
