# 时间管理

## 分钟-分值换算表

$$50 \text{ 分} \div 90 \text{ 分钟} = 1.8 \text{ 分钟/分}$$

每道题的预计用时：

| 题目分值 | 建议用时 | 剩余时间（按顺序作答） |
|---------|---------|---------------------|
| 6 分 | 10–11 分钟 | 79–80 分钟 |
| 7 分 | 12–13 分钟 | 67–68 分钟 |
| 8 分 | 14–15 分钟 | 53–54 分钟 |
| 9 分 | 16–17 分钟 | 37–38 分钟 |
| 10 分 | 18–19 分钟 | 19–20 分钟 |
| 剩余综合题 | 约 20 分钟 | 0–5 分钟检查 |

:::note[时间弹性]
以上为理想情况。实际考试中，简单题可能更快（5–8 分钟），难题可能需要更长时间。核心原则：**单题不超过预算时间的 120%**，超时果断跳过。
:::

## 五阶段应试策略

### Phase 1: 浏览（0–5 分钟）

- **不做任何计算**
- 通读全卷，标记每题的板块和难度
- 确认是否有不熟悉的题型
- 在草稿纸上写下：
  - 题量（6 或 7）
  - 每题的分值
  - 优先级标记（1st / 2nd / 3rd pass）

### Phase 2: 第一轮 — 抢分（5–30 分钟）

- 只做标记为 **简单** 的题目
- 目标：以最快速度拿到确定能拿的分数
- 这些题通常是：
  - 卡方检验（套路固定）
  - 简单的 CRV 计算（求 $k$、求 $E(X)$）
  - 非参数检验的排序部分
- **不要在第一轮思考难题**

### Phase 3: 第二轮 — 攻坚（30–65 分钟）

- 处理标记为 **中等** 的题目
- 每道题最多不超过预算时间的 120%
- 遇到卡顿：
  - 先跳到子问中较易的部分（如 t 检验的 $H_0$ 定义）
  - 写出所有你确定的部分
  - 标记卡住的位置，最后回来处理

### Phase 4: 第三轮 — 扫尾（65–80 分钟）

- 处理标记为 **困难** 的题目
- 此时目标不是拿满分，而是尽可能多拿 **M1** 方法分
- 策略：
  - 写出所有能写的步骤（即使不确定）
  - 尝试子问中的前几问
  - 不要纠结于复杂的代数计算

### Phase 5: 检查（80–90 分钟）

**必检项目（按重要性排序）：**

1. **是否所有题都有作答痕迹？** — 空白 = 0 分
2. **自由度是否正确？** — 最易出错的环节
3. **单尾/双尾选择？** — 与 $H_1$ 对照
4. **配对/独立判断？** — 检查数据特征
5. **Yates 校正是否遗漏？** — $2\times2$ 表必检查
6. **期望频数 $E\ge 5$？** — 否则需合并
7. **所有概率在 [0,1]？** — 超出则必然有误
8. **所有方差 $\ge 0$？** — 负方差必须有误
9. **结论是否有文字解释？** — 在题目情境中说明
10. **是否有单位？** — 如 kg, cm 等

## 卡住决策表

| 情形 | 症状 | 决策 | 措施 |
|------|------|------|------|
| 看不懂题目 | 读了两遍仍不知在问什么 | **跳过** | 先做后面的题，回头再看 |
| 公式记不清 | 知道用什么方法但公式模糊 | **写下部分** | 写 $H_0$、$H_1$ 和公式框架，留空待补 |
| 计算器操作不熟 | 反复按键得不到预期结果 | **手工估算** | 用近似计算代替，或检查统计模式 |
| 积分卡住 | CRV 积分到一半不会了 | **写步骤** | 写出积分的上下限和预期形式，争取 **M1** |
| 自由度不确定 | 两种可能都有道理 | **选保守** | 选较大的自由度（更保守的检验） |
| 时间只剩 10 分钟 | 还有一整题没做 | **抢步骤分** | 写 $H_0$ + 公式 + 代入，不要计算最终值 |
| 卡在某子问 | 子问 (c) 完全没思路 | **做下一子问** | (d) 可能可以用 (c) 的结果（即使没算出） |
| 心算错误 | 发现前面某步计算有误 | **标注继续** | 划掉错误部分，写 "corrected"，继续做后续 |
| 临界值找不到 | 表中没有对应 df | **用保守值** | 取相邻更小的 df（更保守的临界值） |
| 合并方差忘记 | 不知道 $s_p^2$ 公式 | **不合并** | 用 Welch t 检验（不合并方差，自由度用近似） |
| 很难的 PGF | 复合函数求导太复杂 | **写定义** | 写出 $G_{S_N}(t)=G_N(G_X(t))$ 并代入，争取 **B1** |
| 非参数打结处理 | 多个相同值如何排序 | **取平均秩** | 相同值赋予相同秩（平均值），标注修正 |

## 时间浪费预警

| 浪费时间的行为 | 建议替代方案 |
|--------------|------------|
| 反复检查同一道计算 | 做完后先做下一题，最后回查 |
| 在难题上钻研超过预算时间 | 设定 120% 上限计时器 |
| 完美主义写过多解释文字 | 使用简洁的数学语言 |
| 在草稿纸上写完整答案再抄 | 直接在答题簿上书写，保持整洁即可 |
| 频繁使用计算器验证简单手算 | 相信自己的手算，除非有明显矛盾 |
| 花时间画复杂的表格和图示 | 用简洁的矩阵或列表代替 |
